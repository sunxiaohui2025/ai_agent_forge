💡 为什么要三次握手？

TCP 是可靠的全双工通信协议，三次握手确保双方都能发送和接收数据。第一次 SYN 证明客户端能发，第二次 SYN-ACK 证明服务器能收发，第三次 ACK 证明客户端能收。

💡 连接池的优势

连接池避免了频繁创建和关闭 TCP 连接的开销。HTTP/1.1 默认启用 Keep-Alive，允许在一个 TCP 连接上发送多个 HTTP 请求。现代浏览器通常会对同一域名维护 6-8 个持久连接。

TCP 连接状态转换

ESTABLISHED → 连接已建立
FIN_WAIT_1 → 发送 FIN 后
FIN_WAIT_2 → 收到 ACK 后
TIME_WAIT → 收到 FIN 并发送 ACK
CLOSED → 等待 2MSL 后关闭

💡 为什么要四次挥手？

TCP 是全双工通信，双方都需要单独关闭自己的发送通道。TIME_WAIT 状态需要等待 2MSL（最大报文生存时间），确保最后的 ACK 能被对方收到，并处理迟到的报文。
