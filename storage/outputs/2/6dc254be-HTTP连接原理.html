Three-Way Handshake

三次握手流程

客户端与服务器通过三次报文交换完成连接建立，确保双方收发能力正常。

握手过程 · Handshake Process
SYN
第一次握手
客户端发送SYN包
seq = x
请求建立连接
SYN-ACK
第二次握手
服务器回复SYN+ACK
seq = y, ack = x+1
确认收到并请求
ACK
第三次握手
客户端回复ACK
ack = y+1
连接正式建立
为什么需要三次？
第一次确认客户端能发，第二次确认服务器能收能发，第三次确认客户端能收。两次无法确认双方的收发能力都正常。

03 / 03
Four-Way Handshake

四次挥手断开

TCP连接是全双工的，双方需要独立关闭各自的数据传输通道，因此需要四次报文交换。

挥手过程 · Termination Process
FIN
第一次挥手
主动方发送FIN
表示不再发送数据
ACK
第二次挥手
被动方回复ACK
确认收到断开请求
FIN
第三次挥手
被动方发送FIN
也不再发送数据
ACK
第四次挥手
主动方回复ACK
连接完全关闭
为什么断开比建立多一次？
建立连接时，服务器可将SYN和ACK合并为SYN-ACK一次性回复。而断开时，被动方可能还有数据要发，必须等待数据发完才能发FIN，因此ACK和FIN不能合并。
